# Sprint 0 — Collection Model Expansion

## Specs Covered

- `docs/_specs/curatorial-enrichment/spec.md`

## Goal

Make the collection model strong enough to govern future object additions.

## Scope

### In

- expand object-role guidance for threshold, proof, cast, scale, and atmosphere objects
- expand room-role guidance for primary and supporting spaces
- define when new objects or additions are justified
- keep the model lightweight and repository-grounded

### Out

- redesigning live exhibit pages without a clear model-driven need
- adding speculative content categories with no use in the current museum
- changing unrelated visual-system files

## Tasks

1. Expand object-role guidance.
   - distinguish threshold, proof, cast, scale, and atmosphere objects clearly
   - define when a new object is justified
2. Expand room-role guidance.
   - explain which rooms are primary and which are supporting
   - define what kinds of additions belong in each room type

## Files to touch

- `docs/content/collection-model.md`

## Acceptance criteria

- object roles are clearer
- room roles are clearer
- the model can guide future additions without improvisation
- the model limits random or decorative additions
- the guidance remains lightweight and grounded in the current repository

## Verification checklist

- [ ] confirm the collection model can guide a future addition without improvisation
- [ ] ensure the model remains lightweight and repository-grounded
- [ ] verify object-role guidance is specific enough to use in future sprints
- [ ] verify room-role guidance distinguishes primary and supporting spaces
