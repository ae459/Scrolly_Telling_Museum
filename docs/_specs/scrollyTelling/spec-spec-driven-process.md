# Purpose

Maintain a lightweight, spec-driven workflow for all major curatorial and visual changes.

## What Done Looks Like

- Every major change starts with a spec in the appropriate folder.
- Specs are small, focused, and actionable.
- Sprint docs reference relevant specs.

## Constraints / Non-Goals

- Does not require specs for trivial copy edits or bugfixes.
- Does not duplicate existing specs in other domains.
