# Sprint 1 — Room System And Object Surfaces

## Specs Covered

- `docs/_specs/site-exhibition/spec.md`

## Goal

Define the museum's page taxonomy and collection logic so future additions remain curator-driven.

## Scope

### In

- define chronology rooms versus companion rooms
- define object roles such as threshold, proof, cast, scale, and atmosphere objects
- preserve visual discipline for cards, labels, and caption systems
- document how new room or object proposals should be evaluated

### Out

- adding large amounts of new exhibit content without curatorial justification
- redesigning the full site visual system
- changing unrelated process documents

## Tasks

1. Define room roles.

- identify chronology rooms vs companion rooms
- state which surfaces are primary and which are subordinate

2. Define object roles.

- identify threshold, proof, cast, scale, and atmosphere objects
- keep additions tied to room logic

3. Protect visual discipline.

- preserve consistent card, label, and caption systems
- require visual review for new object media

## Files to touch

- `docs/content/collection-model.md`
- relevant room pages in `docs/` when updates are needed

## Acceptance criteria

- room taxonomy is explicit
- object roles are explicit
- future additions have a curatorial frame before implementation
- the model strengthens chronology rather than flattening it
- visual discipline is treated as part of content quality

## Verification checklist

- [ ] check that new object or room proposals can be mapped to the model
- [ ] confirm the model strengthens the chronology rather than flattening it
- [ ] verify room roles are distinguishable in the documentation
- [ ] verify object roles are concrete enough to guide future edits

## Completion evidence

- URL: repository paths to the updated collection-model and any touched room pages
- Screenshots: optional side-by-side capture of room/object guidance in markdown preview
- PR / commit: link or commit hash when the sprint is implemented
