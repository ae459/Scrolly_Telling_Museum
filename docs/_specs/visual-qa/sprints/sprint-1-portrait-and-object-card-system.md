# Sprint 1 — Portrait And Object Card System

## Specs Covered

- `docs/_specs/visual-qa/spec.md`

## Goal

Align the portrait cards and non-comic object cards into a shared visual family.

## Scope

### In

- standardize portrait nameplates and spacing
- standardize object-card labels and borders
- align portrait and object cards with the comic-card system where appropriate
- verify consistency in the homepage and people/studios views

### Out

- redesigning the core comic-card family beyond compatibility needs
- adding new portraits or objects without a separate content need
- changing unrelated room layouts

## Tasks

1. Standardize portrait nameplates.
   - centered labels
   - consistent title and subtitle spacing
   - consistent portrait frame proportions
2. Standardize object-card labels.
   - object titles and sublabels align consistently
   - border and spacing systems feel related to the comic cards

## Files to touch

- portrait SVGs in `docs/media/`
- object SVGs such as `docs/media/bullpen-desk.svg` and `docs/media/mcu-helmets.svg`

## Acceptance criteria

- portrait cards feel unified
- object cards feel unified
- no one-off label system remains
- the portrait and object cards visibly belong to the same exhibition family

## Verification checklist

- [ ] confirm touched SVGs report no errors
- [ ] preview the people room and homepage for consistency
- [ ] verify portrait labels share centered alignment and consistent spacing
- [ ] verify object-card borders and labels relate cleanly to the comic-card system

## Completion evidence

- URL: repository paths to the updated portrait and object SVGs
- Screenshots: captures of the homepage and people/studios room showing the aligned card family
- PR / commit: link or commit hash when the sprint is implemented
